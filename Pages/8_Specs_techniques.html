Wassel'lia – Spécifications techniques & architecture

Documentation technique – Cloud, PWA, base de données, IA, sécurité – Confidentiel
Vue d’ensemble de la plateforme

Wassel'lia est une application web PWA (Progressive Web App) full‑stack, mobile‑first, qui permet aux clients de soumettre une demande de devis de déménagement, aux modérateurs de générer un devis assisté par IA, et aux prestataires de suivre leurs missions. L’architecture est cloud‑native, serverless, avec une base de données PostgreSQL (Supabase), des edge functions Deno, et un frontend React/Vite.

IA & automatisation des devis

Génération du devis (IA)

L’edge function `generate-quote` reçoit les données de la demande (type logement, formule, inventaire, adresses, étages, distance). Elle appelle l’API Lovable / OpenAI avec un prompt structuré pour calculer :
– Forfait base (studio, 2 pièces, 3+, maison)
– Supplément distance (5 DH/km au‑delà de 10 km)
– Supplément étages sans ascenseur (50 DH/étage)
– Majoration formule clé en main (+40%)
– TVA 20% et commission Wassel'lia (25%)
L’IA retourne un JSON détaillé (justificatif ligne à ligne).

Génération PDF (pdf‑lib)

Une fois le devis validé par le modérateur, `generate-quote-pdf` crée un PDF professionnel avec mentions légales, logo, conditions générales, et le stocke dans le bucket `quotes`. Un lien signé est envoyé au client par WhatsApp.

Communication WhatsApp

Actuellement, l’envoi du devis se fait via un lien `wa.me/` manuel. En phase 2, intégration de Twilio WhatsApp Business API pour l’envoi automatique, les boutons “CONFIRMER” / “ANNULER” et le webhook de confirmation.

Processus 48h automatisé

Des triggers PostgreSQL et des fonctions planifiées (cron) surveillent les délais. Si une demande reste bloquée plus de 48h, une notification est envoyée au modérateur et le client est prévenu.

Sécurité & conformité

Authentification & accès

• JWT avec expiration courte (1h).
• MFA TOTP obligatoire pour tous les comptes admin (Google Authenticator).
• Vérification des mots de passe compromis (HIBP).
• Signups publics désactivés – création manuelle via invitation.

Protection des données

• TLS 1.3 pour toutes les communications.
• Chiffrement AES‑256 au repos (Supabase).
• RLS : isolation stricte entre clients, modérateurs et admins.
• Conformité loi 09‑08 (CNDP) : déclaration, politique de confidentialité, bandeau cookies.

📜 Contrats électroniques

Les contrats signés par les clients (pad de signature ou case à cocher + nom + date) sont stockés en PDF dans un bucket privé, avec horodatage. La signature électronique est réputée fiable sous la loi 53‑05.

Sauvegarde & continuité

• Sauvegarde quotidienne automatique de la base (7 jours de rétention).
• RTO < 4 heures, RPO < 1 heure.
• Hébergement Supabase (UE) – redondance régionale.

Modérateur ouvre dashboard → sélectionne la demande → clique "Générer le devis" 3. Edge function `generate-quote` appelle Lovable IA + Google Distance Matrix → calcule prix et génère justificatif JSON 4. Modérateur visualise l’aperçu → clique "Envoyer par WhatsApp" 5. Edge function `generate-quote-pdf` crée PDF → stocke dans bucket → génère URL signée (7j) 6. Envoi via lien `wa.me/` (manuel) ou API Twilio (automatique) → client reçoit devis 7. Client répond CONFIRMER (webhook) → statut passe à 'quote_accepted' 8. Client finalise paiement / signature contrat → statut 'contract_signed' → processus 48h démarre
