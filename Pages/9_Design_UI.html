Logo & iconographie

Logo monochrome brun (icône camion + texte « Wassel'lia »). Favicon assorti. Icônes vectorielles (FontAwesome / Lucide) utilisées pour toutes les actions – aucun émoji. La barre de navigation affiche uniquement l’icône (pas de texte long) sur mobile.

Tonalité & style

Chaleureux, professionnel, marocain moderne. Pas de fausse minimaliste : cartes avec ombres douces, coins arrondis 2xl, motifs de fond subtils (points, vagues à faible opacité). Transitions fluides (0.2s).

Composants d’interface (shadcn/ui personnalisé)

Boutons

Primary fond #84593D, texte blanc, padding 0.75rem 1.5rem, border-radius 2rem.
Secondary fond transparent, bordure #84593D, texte #84593D.
Ghost sans fond ni bordure, apparaît au survol.
Désactivé opacité 0.5, curseur non autorisé.

Formulaires & inputs

Champs avec bordure #f0e5dd, coins 0.75rem, padding 0.75rem, focus avec anneau #D9A441. Labels en Inter Medium, messages d’erreur en rouge (#b13e3e) en petit corps. Validation Zod en temps réel.

Cartes (cards)

Fond blanc, bordure 1px #f0e5dd, box‑shadow 0 12px 25px rgba(0,0,0,0.04), coins 1.5rem, padding 1.5rem. Au survol, ombre plus marquée et translation Y (-2px).

Modales & overlays

Centrées, fond overlay flou (backdrop‑filter), fermeture par clic externe. Coins 1.5rem, padding 1.5rem. Utilisées pour la signature électronique, la confirmation d’annulation, et l’aperçu du devis.

Timeline de suivi (statuts)

Composant vertical affichant les 5 étapes : Demande → Devis accepté → Inspection → Déménagement en cours → Terminé. Étape active : icône pleine #84593D, étape future : grise. Connecteur gris.

Notifications & toasts

Messages temporaires en bas à droite, fond #5A3A1F, texte blanc, icône de succès/erreur. Disparaissent automatiquement après 4 secondes.

Responsive & accessibilité

Mobile‑first (breakpoints)

Base : ≤375px (iPhone SE).
sm 640px · md 768px · lg 1024px · xl 1280px.
Navigation : menu burger sur mobile, barre horizontale sur desktop.
Formulaires : champs pleine largeur sur mobile, deux colonnes sur desktop.

Accessibilité (WCAG 2.1 AA)

• Contraste des couleurs vérifié (ratio > 4.5:1).
• Attributs `aria-label` sur toutes les icônes.
• Navigation clavier (Tab, Enter, Escape).
• Tailles de police relatives (rem).
• Messages d’erreur clairs pour les formulaires.

Performances PWA

• Service worker avec mise en cache des assets statiques (stratégie stale‑while‑revalidate).
• Manifest.json pour installation sur smartphone.
• Lazy loading des images et des composants lourds.
• Score Lighthouse visé > 90 sur performance, accessibilité, SEO.

Gestion des erreurs & états vides

• Pages 404, 500 personnalisées.
• États de chargement (squelettes / spinners).
• Messages explicites quand aucune demande trouvée, pas de prestataire disponible, etc.

Flux utilisateur – Client particulier
1. Client arrive sur la page d’accueil → lit la proposition de valeur → clique « Demander un devis ». 2. Remplit le formulaire en 5 étapes (type, formule, inventaire, adresses, coordonnées). 3. Soumet → voit une page de confirmation « Votre demande a été transmise, vous recevrez un devis par WhatsApp sous 24h. » 4. Le modérateur génère le devis IA → envoie le lien WhatsApp. 5. Client reçoit le message → clique sur le lien → visualise le devis (PDF) → répond CONFIRMER. 6. Client reçoit un lien de paiement / signature → signe le contrat électronique → finalise. 7. Client reçoit un rappel la veille du déménagement. 8. Le jour J, le prestataire arrive → client peut suivre le statut dans la page de suivi. 9. Après le déménagement, client reçoit un lien d’évaluation (note sur 5 + commentaire).
Évolutions UI/UX prévues (2026‑2027)

Suivi GPS en temps réel

Intégration de Leaflet / Mapbox pour afficher la position du prestataire sur une carte pendant le déménagement.

Chat intégré (WhatsApp embarqué)

Widget de messagerie directe depuis l’application (sans quitter le navigateur) via WhatsApp Business API.

Personnalisation du devis par IA conversationnelle

Assistant IA (chatbot) qui guide le client dans la description de son inventaire, posant des questions sur les objets fragiles, l’accès, etc.

Mode sombre (dark mode)

Basé sur les préférences système, avec adaptation de la palette (brun sur fond foncé).

Synthèse – Pourquoi l’UI/UX de Wassel'lia est un atout
  • ✅ Interface épurée mais chaleureuse, inspirée de Tiptapp et Lalamove – confiance immédiate.
  • ✅ Mobile‑first : plus de 80% des utilisateurs réservent sur smartphone – l’expérience est fluide.
